# Eval 測試案例（草稿）— 管理辦法小幫手

> 建置 `assistant/rag.py` 時轉成 `eval/tests.json` 跑回歸。標準答案依 `corpus/sources/` 生效中版原文。
> 每題驗三件事：①路由對（撈到對的辦法）②答案+引用對 ③反唬爛（沒寫的別掰）。

> ⚠️ **合成辦法 caveat（2026-06-26）**：本 eval 所有「標準答案」綁定 `sources/` 下的**合成辦法**（AI 生成、非真實公司文件；真實辦法要等本套落地後才匯入）。落地換真辦法時**分三層**處理：
> - **內容層（必換）**：標準答案依賴辦法**具體內容/實務**者（金額、日數、門檻、H 題「出差中請假手續」等）→ 一律以真實辦法重新校準。
> - **能力層（不隨資料變、現在就驗）**：eval 題揭露的**能力需求**（H 的「答程序要補撈同辦法關聯條文」、交叉引用沿圖撈、版本按日期守門、反唬爛）與辦法真假無關，是架構該具備的能力，PoC 用合成辦法當代理先驗——**不可因「等真辦法」而延後**。
> - **架構層（已知風險，user 2026-06-26 提示）**：若落地後真實辦法的**結構**（條文交互、版本鏈、跨辦法引用樣態）與合成假設不同、**且牽動引擎架構**（攝取/檢索/答題），則必須回頭修架構。**已知風險、非設計疏漏。**

## A. 答得出 + 引用對（基本檢索）
- **問**：出差住宿費上限多少？
- **應路由到**：出差管理辦法（現行）
- **標準答案**：國內每人每晚上限 2,000 元（實報實銷）。
- **引用**：出差管理辦法 第三條。

## B. 查無 → 誠實說查無（反唬爛）
- **問**：公司有沒有寵物友善政策（可以帶寵物上班嗎）？
- **應路由到**：無相關辦法（或撈到但內容不含）
- **標準答案**：現行辦法查無相關規定。**不得編造**。

## C. 版本陷阱 → 回現行版（版本正確性命門）
- **問**：採購金額多少以上要三家比價？
- **應路由到**：採購管理辦法 **現行 v2.0**（廢止 v1.0 不得進路由）
- **標準答案**：**10 萬元**以上。**不可答 5 萬**（5 萬是已廢止 v1.0 的舊規定）。
- **引用**：採購管理辦法（現行）第二條。

## D. 綜合題：語意路由 + 引用 + 反唬爛 + 防跨辦法污染（★最嚴）
- **問**：如果家裡突然有急事，我明天沒辦法去公司怎麼辦？
- **考點**：
  1. **零關鍵字語意路由**：問題無「請假/事假」字眼 → 仍需路由到「請假管理辦法／事假」。（純關鍵字會 miss，是 LLM 路由的試金石。）
  2. **答案+引用**：屬事假；經差勤系統**事先申請**、主管核准；事假每年上限 14 日、不給薪。引用 請假管理辦法 第四條（事假）+ 第五條（程序）。
  3. **反唬爛**：請假辦法**未規定「緊急情況事後補假」程序** → 須**誠實指出辦法未明文**（可建議先聯繫主管，但註明超出辦法明文），**不得編造**事後補假條款。
  4. **防跨辦法污染**：出差管理辦法第二條有「緊急出差得事後補單」——**不得**把出差的緊急條款錯套到請假上。
- **失敗模式**：①漏路由到請假辦法（純關鍵字會犯）②憑空生「可電話通知事後補假」③把出差緊急補單套到請假。

## E. 交叉引用（A 援引 B）→ 跟著連結走、兩份都涵蓋（★引用圖測試）
- **前置（建置時要在 sample_docs 埋一條交叉引用）**：於 出差管理辦法 加「第六條　出差期間如需請假，依**請假管理辦法第四條**辦理。」
- **問**：我出差到一半，家裡有事要請假，怎麼算？
- **應路由到**：出差管理辦法（第六條）→ **沿引用圖連到** 請假管理辦法 第四條。
- **標準答案**：出差期間請假依請假辦法辦理，屬事假；事假每年上限 14 日、不給薪。**同時引用** 出差管理辦法 第六條 **＋** 請假管理辦法 第四條。
- **考點**：①攝取時把「依請假辦法第四條」抽成連結 ②取原文時自動把請假第四條一起撈 ③答案涵蓋並引用兩份。
- **失敗模式**：①只引用出差第六條、把「依請假辦法辦理」原句丟回去當答案（沒解析引用）②未取到請假第四條卻**編造**它的內容（反唬爛）③跟連結無限跳 / 撞循環。

## F. 對話追問（multi-turn → 問題改寫）
- **前置（建置時補 sample_docs）**：請假管理辦法加「婚假 8 日」「補休（加班補休）」條文。
- **對話**：① 問「婚假可以請多久？」→ 答「8 天」（引用婚假條）。② 緊接問「**那補休呢？**」。
- **應行為**：把追問**改寫成自足問題**「補休的規定是什麼？」→ 路由到補休條文 → 答**補休**規定。
- **失敗模式**：①「那補休呢？」原樣送進去 → 路由看不懂、答非所問或查無 ②誤把它當「婚假」延續、又答婚假。

## G. 時間版本（未來生效日 → 答生效中版、預告新版）★時間盲區
- **前置（建置時補 sample_docs）**：生育津貼補助辦法 **v1**（生效中，金額 X）+ **v2**（生效日設為「今天之後」如 2026-07-01，金額 Y；v1 失效日設為 v2 生效日−1）。
- **問（參考日＝今天，設為 v2 生效前，如 6/26）**：現在生小孩補助多少？
- **標準答案**：答 **v1 金額 X**（生效中版）；**不可**答 v2 金額 Y（v2 尚未生效）；**並主動預告**「新版 v2 將自 〔生效日〕起實施，屆時為 Y」。引用生育津貼辦法 v1。
- **失敗模式**：①答未生效的 v2 金額（會釀勞資糾紛）②只答 v1、漏掉預告 v2。
- **延伸題**：問「**7 月之後**生小孩補助多少？」→ 參考日換成 7 月 → 答 **v2 金額 Y**。

## H. 出差期間請事假的「手續」（交叉引用 ＋ 程序條文補全 ＋ 防出差程序污染）★比 E 更狠
- **問**：請問出差的時候如果遇到家裡有事要請事假，手續要怎麼辦？
- **應路由到**：出差管理辦法（第六條）→ 沿引用圖連到 請假管理辦法 第四條（事假）；**且因問「手續」須補撈** 請假管理辦法 **第五條（請假程序）**。
- **標準答案**：出差期間請假依請假管理辦法辦理，屬事假（出差第六條→請假第四條）。手續依**請假辦法第五條**：應**事先於差勤系統提出申請、經主管核准後生效**；未經核准擅自不到以曠職論處。**引用** 出差第六條 ＋ 請假第四條 ＋ 請假第五條。
- **考點**：
  1. **交叉引用**：出差第六條 → 請假第四條（屬事假）。
  2. **★程序條文補全**：問的是「手續」，答案在請假**第五條（程序）**——但出差第六條的引用**只指向第四條**。系統須能補撈同辦法的程序條文，**不能只停在引用圖指到的那一條**。
  3. **★防跨辦法污染**：出差**第二條**有「緊急出差**得事後補單**」的程序 → **不得**錯套成「請假可事後補假」。請假第五條是「**事先**申請」，且請假辦法**無**「緊急事後補假」條款。
  4. **反唬爛**：辦法未規定「出差中請假」的特殊簡化手續 → 就是一般請假程序，不得編造特殊流程。
- **失敗模式**：①只把「依請假辦法辦理」原句丟回、不給具體手續 ②把出差第二條「事後補單」套成「可事後補假」（跨辦法污染）③只引第四條講「14日不給薪」、漏第五條沒答到「手續」（答非所問）④編造「出差中請假走特殊流程」。
- **與 E 的差別**：E 問「怎麼**算**」（答案在第四條，引用圖直接指到）；H 問「**手續**」（答案在第五條，**引用圖沒指到、要補撈**）＋ 多一層出差第二條程序污染陷阱。

## B2. 冷門假別 → 誠實查無（反唬爛·假別不窮舉的另一面）
- **問**：喪假可以請幾天？
- **應路由到**：請假管理辦法（靠「喪假 ≈ 假別 ≈ 請假辦法」語意 generalize 路由，**非** tags 內有「喪假」字串）。
- **標準答案**：**現行請假管理辦法未規定喪假日數**，誠實指出辦法未明文、**不得編造**。
- **考點**：①路由仍要命中請假辦法（即使 tags 未列喪假）②內文無喪假條文 → 誠實查無，不因「聽過喪假」就生一個日數。
- **失敗模式**：①漏路由（以為 tags 沒喪假就找不到）②編造喪假日數（如「三日」）。
